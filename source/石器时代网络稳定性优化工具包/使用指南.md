# 石器时代网络稳定性优化工具包 - 使用指南

## 📋 概述

本工具包专门为解决石器时代脚本开发中的网络波动和卡顿问题而设计。基于用户明确要求的最高优先级需求：
1. **解决网络波动和卡顿问题** - 脚本经常因网络问题卡住
2. **增强稳定性和容错性** - 确保脚本能平稳运行
3. **迅速平稳完成** - 减少中断，提高完成率

## 🔧 核心模块详解

### 1. 网络波动检测器 (`网络波动检测器.py`)

#### 功能
- 实时监控网络连接状态
- 自动检测网络波动和中断
- 智能判断网络恢复时间
- 提供网络状态报告和预警

#### 配置项
```json
{
  "check_interval": 5.0,          # 检查间隔(秒)
  "ping_timeout": 3.0,           # ping超时时间(秒)
  "event_thresholds": {
    "high_latency": 200,         # 高延迟阈值(ms)
    "packet_loss_warning": 10,   # 丢包警告阈值(%)
    "connection_timeout": 10     # 连接超时阈值(秒)
  }
}
```

#### 使用方法
```python
from 网络波动检测器 import NetworkMonitor

# 初始化
monitor = NetworkMonitor("配置/网络监控配置.json")

# 开始监控
monitor.start_monitoring()

# 获取当前状态
status = monitor.get_current_status()
print(f"网络状态: {status}")

# 停止监控
monitor.stop_monitoring()
```

### 2. 智能重试系统 (`智能重试系统.py`)

#### 功能
- 智能判断何时需要重试
- 自适应重试策略（指数退避、随机延迟等）
- 重试成功率预测和优化
- 与ASSA脚本无缝集成，自动恢复执行

#### 重试策略
- **指数退避**: 经典指数退避算法
- **固定间隔**: 固定时间间隔重试
- **随机退避**: 随机时间退避避免同步
- **自适应**: 根据历史成功率调整
- **混合策略**: 组合多种策略

#### 使用方法
```python
from 智能重试系统 import IntelligentRetrySystem, RetryTask, RetryStrategy

# 初始化
retry_system = IntelligentRetrySystem("配置/重试系统配置.json")

# 定义需要重试的函数
def risky_operation(param1, param2):
    # 可能失败的操作
    return result

# 创建重试任务
task = RetryTask(
    task_id="task_001",
    task_name="危险操作",
    function_to_retry=risky_operation,
    function_args=(arg1, arg2),
    max_attempts=5,
    strategy=RetryStrategy.EXPONENTIAL_BACKOFF
)

# 执行带重试的任务
try:
    result = retry_system.execute_with_retry(task)
    print(f"操作成功: {result}")
except Exception as e:
    print(f"所有重试都失败: {e}")
```

### 3. 防卡机制 (`防卡机制.py`)

#### 功能
- 实时监控脚本执行状态
- 检测卡顿和死锁
- 自动恢复和继续执行
- 位置验证和异常坐标纠正

#### 卡顿类型
- **位置卡顿**: 位置长时间不变
- **网络卡顿**: 网络响应超时
- **资源卡顿**: 资源等待超时
- **逻辑卡顿**: 逻辑循环卡住
- **界面卡顿**: 界面无响应

#### 使用方法
```python
from 防卡机制 import AntiStuckMechanism

# 初始化
anti_stuck = AntiStuckMechanism("配置/防卡机制配置.json")

# 开始监控（通常在脚本主循环中）
anti_stuck.start_monitoring()

# 在脚本关键位置添加检查点
def move_to_position(x, y):
    # 记录位置
    anti_stuck.record_position((x, y), "当前地图")
    
    # 执行移动操作
    # ...
    
    # 验证是否到达
    if not anti_stuck.verify_position((x, y), tolerance=5):
        print("位置验证失败，可能卡顿")
        # 触发恢复机制
        recovery_action = anti_stuck.detect_and_recover()
        print(f"恢复动作: {recovery_action}")

# 停止监控
anti_stuck.stop_monitoring()
```

## 🔄 集成到ASSA脚本

### 基本集成模式

#### 模式1: 包装现有脚本
```python
# 原ASSA脚本
def original_assa_script():
    # 原有的ASSA代码
    pass

# 包装版本
def enhanced_assa_script():
    from 智能重试系统 import IntelligentRetrySystem, RetryTask
    
    retry_system = IntelligentRetrySystem()
    
    task = RetryTask(
        task_id="assa_main",
        task_name="ASSA主脚本",
        function_to_retry=original_assa_script,
        max_attempts=3
    )
    
    return retry_system.execute_with_retry(task)
```

#### 模式2: 嵌入监控点
```python
from 防卡机制 import AntiStuckMechanism
from 网络波动检测器 import NetworkMonitor

class EnhancedASSA:
    def __init__(self):
        self.anti_stuck = AntiStuckMechanism()
        self.network_monitor = NetworkMonitor()
        
    def run(self):
        # 启动监控
        self.network_monitor.start_monitoring()
        self.anti_stuck.start_monitoring()
        
        try:
            # 主脚本逻辑
            self._main_loop()
        finally:
            # 停止监控
            self.anti_stuck.stop_monitoring()
            self.network_monitor.stop_monitoring()
    
    def _main_loop(self):
        while True:
            # 检查网络状态
            if self.network_monitor.current_status == "DISCONNECTED":
                print("网络断开，暂停执行")
                self.anti_stuck.handle_stuck("网络断开")
                continue
            
            # 执行ASSA操作
            self._perform_assa_operation()
            
            # 记录位置防止卡顿
            current_pos = self._get_current_position()
            self.anti_stuck.record_position(current_pos)
```

## 🚨 故障排除

### 常见问题

1. **网络监控无法启动**
   - 检查ping3库是否安装: `pip install ping3`
   - 检查防火墙设置，允许ICMP协议
   - 尝试使用不同的检测目标

2. **重试系统效率低**
   - 调整重试策略参数
   - 检查成功条件函数是否正确
   - 查看重试历史记录分析失败原因

3. **防卡机制误报**
   - 调整检测阈值
   - 添加白名单位置
   - 优化位置验证逻辑

4. **性能问题**
   - 减少监控频率
   - 限制历史数据大小
   - 关闭不必要的模块
